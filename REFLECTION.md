# Reflection

## Decisions
One of the most important decisions I made during this project was designing the website to feel beginner-friendly and welcoming instead of overly clinical or technical. Since many people become nervous when talking about dental care, I wanted the website to feel approachable and easy to understand. I focused on using simple language, educational visuals, and organized sections so users could quickly learn about cavities, gingivitis, and periodontal disease without feeling overwhelmed.

Another major decision I made was redesigning the visual appearance after receiving peer feedback. My peers suggested improving the color scheme and making the site feel more visually engaging. Based on that feedback, I added the purple gradient header, adjusted the homepage layout, and included orange hover effects on the tabs to give the website more contrast and personality. I wanted the website to feel polished while still maintaining a calm healthcare aesthetic.

## What worked
One thing that worked really well was the final overall appearance and organization of the website. Compared to Project 01, the website became much cleaner, more polished, and easier to navigate. The tabs helped separate information into manageable sections, and the educational images and videos made the site feel more interactive and engaging.

I was especially happy with how the gingivitis and periodontal disease sections turned out because the visuals helped explain concepts in a more patient-friendly way. The hover effects, rounded sections, gradients, and spacing improvements also helped the website look more professional and modern. I also successfully learned how to deploy my project through GitHub Pages, which felt like a major accomplishment because I had never used GitHub before this course.

## What I would do differently
If I could restart this project, I would spend more time planning my file organization and layout before building the website. A lot of my challenges came from small coding mistakes and disorganization, such as forgetting to close div tags, placing files in the wrong folders, or accidentally breaking sections of the website while editing CSS. Since I am still a beginner, troubleshooting those mistakes sometimes took a long time.

I would also give myself more time to experiment with responsive design and accessibility features. If I continued improving the project, I would add more educational sections, animations, and interactive features like quizzes or clickable diagrams to make the experience even more engaging for users.

## What I learned
This project was honestly one of the hardest projects for me because I am still a beginner and had never used GitHub or deployed a website before this class. A large part of the process involved trial and error, troubleshooting, and learning from small coding mistakes. There were multiple times where my website would stop working because I forgot to close a div tag, misplaced code, or accidentally changed formatting in my CSS. Although frustrating at times, those moments helped me better understand how HTML and CSS work together and taught me how important organization and attention to detail are in web development.

I also learned how to use GitHub Pages, upload files correctly, manage repositories, and troubleshoot deployment issues. In the beginning, even simple tasks like organizing images or getting videos to display properly felt overwhelming. Over time, I became much more comfortable navigating GitHub and editing my code.

Peer feedback also played a major role in improving my project. Several peers suggested improving the color scheme and making the design feel more visually engaging. Based on that feedback, I redesigned parts of the homepage by adding the purple gradient header, adjusting the layout, and using orange hover effects on the navigation tabs to create a brighter pop of color and a more interactive experience. I think these design changes helped the website feel more polished, modern, and welcoming for users.

Overall, this project pushed me outside of my comfort zone, but it also helped me gain confidence in both coding and creative problem-solving. Even though the process was difficult at times, I am proud of how much the final website improved from my original version.
